01반복문 — while · for

반복문은 같은 코드를 여러 번 실행할 때 사용한다. 조건이 true인 동안 계속 반복하고, false가 되면 멈춘다.

while 문

조건이 true인 동안 계속 반복한다. 반복 횟수를 미리 알 수 없을 때 사용한다.

let w = 0;
while (w < 5) {
  console.log(w);  // 0 1 2 3 4
  w++;             // 증감식 — 없으면 무한루프!
}
⚠ w++처럼 변수를 변화시키는 증감식이 없으면 조건이 절대 false가 되지 않아 무한루프가 된다.

for 문

초기식 · 조건식 · 증감식을 한 줄에 묶어 쓴다. 반복 횟수가 명확할 때 편리하다.

for (let i = 0; i < 5; i++) {
  console.log(i);  // 0 1 2 3 4
}
//  ↑초기식   ↑조건식  ↑증감식
💡 const i는 값 변경이 불가해 오류가 난다. let i를 사용해야 한다.

배열 순회 — break · continue

배열 인덱스는 0부터 시작하므로 조건은 arr.length 미만(<)으로 설정해야 한다.

const arr = ['아침', '간식', '점심', '간식', '저녁'];

for (let i = 0; i < arr.length; i++) {
  if (arr[i] === '간식') continue;  // 해당 반복 건너뜀
  console.log(arr[i]);
  if (arr[i] === '점심') break;    // 반복 전체 종료
}
// 출력: 아침  점심
continue
  • 해당 반복만 건너뛰고 다음으로 진행
break
  • 반복 전체를 즉시 종료

역방향 순회 · 중첩 for 문

// 역방향
for (let i = arr.length - 1; i >= 0; i--) {
  console.log(arr[i]);
}

// 중첩 for — 2차원 배열(행·열) 탐색
const matrix = [
  [1, '윌리엄', '바나나를 좋아함'],
  [2, '김자바', '김을 좋아함'],
];
for (let i = 0; i < matrix.length; i++) {
  for (let j = 0; j < matrix[0].length; j++) {
    console.log(matrix[i][j]);
  }
}
⚠ 중첩이 깊을수록 실행 횟수가 기하급수적으로 늘어난다(O²). 꼭 필요할 때만 사용할 것.

02for...in & for...of

배열과 객체를 순회하는 전용 반복문. 꺼내는 것이 키(key)냐 값(value)이냐가 핵심 차이다.

for...in — 키(key) 순회

객체의 속성 이름(키)을 하나씩 꺼낸다. 배열에 쓰면 인덱스가 문자열로 나온다.

const obj = { name: '윌리엄', job: '반찬가게 사장', salary: 300_000 };

for (const key in obj) {
  console.log(key, obj[key]);
}
// name 윌리엄 / job 반찬가게 사장 / salary 300000
💡 숫자에 _를 사용하면 가독성 구분자로 쓸 수 있다. 300_000 = 300000

for...of — 값(value) 순회

const arr = ['연어', '아보카도', '스테이크', '렌틸콩'];

for (const v of arr) {
  console.log(v);  // 연어 / 아보카도 / 스테이크 / 렌틸콩
}

// 일반 객체에 for-of 바로 쓰면 TypeError!
// → Object.values() 나 Object.keys() 로 먼저 변환
for (const v of Object.values(obj)) {
  console.log(v);
}
구분for...infor...of
꺼내는 것키(key), 인덱스값(value)
사용 대상객체 속성 탐색배열·문자열 순회
객체에 직접 사용✅ 가능❌ TypeError (변환 필요)
배열 인덱스 타입문자열 "0", "1"...해당 없음 (값 직접)

03함수 기본 — 스코프 · return

함수는 코드를 묶어두었다가 필요할 때 호출하는 블록이다. 함수 안에서 선언된 변수는 함수 밖에서 접근할 수 없다.

전역 변수 vs 지역 변수

let a = 0;  // 전역 변수 — 어디서든 접근 가능

function myFun() {
  console.log(a);              // 전역 변수 읽기 가능
  a++;                         // 전역 변수 수정도 가능
  let b = '나는 함수 안 변수';  // 지역 변수
  return b;
}

myFun();
console.log(a);    // 1
// console.log(b); // ReferenceError — 지역 변수는 밖으로 못 나옴
❌ 함수 안의 지역 변수는 밖에서 절대 접근 불가. 꺼내려면 반드시 return을 사용해야 한다.

변수 우선순위 — 가까운 스코프가 이긴다

let a = 0;  // 전역

function myFun2(a) {   // 매개변수 a — 함수 스코프
  {
    let a = 1000;       // 블록 스코프 a
    console.log(a);    // 1000
  }
  return a * 2;        // 매개변수 a 사용 → 100 * 2
}

console.log(myFun2(100)); // 200
우선순위스코프 종류예시
1순위 (가장 강함)블록 스코프{ let a = 1000 }
2순위매개변수 (함수 스코프)function f(a)
3순위전역 변수let a = 0 (최상단)

early return 패턴

function myFun3() {
  const r = Math.random();
  if (r > 0.9) return [1, 2, 5];     // 배열 반환
  if (r > 0.5) return { a: 10 };      // 객체 반환
  return '동전던지기 실패';              // 문자열 반환
}
✅ return을 만나면 함수가 즉시 종료되므로, 그 뒤에 else가 필요 없어진다. 이것을 early return 패턴이라 한다.

04매개변수 — 기본값 · rest · 객체 전달

매개변수(parameter)는 함수 선언부에 쓰는 변수 이름이고, 인자(argument)는 호출할 때 실제로 넣는 값이다.

인자 개수가 달라도 오류가 없다

function f1(a, b, c) {
  console.log(a * 2, b * 2, c * 2);
}

f1();           // NaN NaN NaN   (undefined * 2 = NaN)
f1(1, 2);       // 2 4 NaN       (c = undefined)
f1(1, 2, 3);    // 2 4 6
f1(1, 2, 3, 4); // 2 4 6  (4는 무시)

기본값 파라미터

function f2(a = 120, b = 150) {
  return a * b;
}

console.log(f2());     // 18000  (120 * 150)
console.log(f2(100));  // 15000  (100 * 150)
💡 기본값은 인자가 undefined일 때만 사용된다. null을 넣으면 기본값이 적용되지 않는다.

나머지 파라미터 (...rest)

function f4(a, ...b) {
  console.log('a', a);
  console.log('b', b);  // b는 항상 배열
}

f4(1);               // a: 1  /  b: []
f4(1, 2, 3, 4, 5);  // a: 1  /  b: [2,3,4,5]
arguments
  • 유사 배열 (배열 메서드 못 씀)
  • 화살표 함수에서 사용 불가
...rest
  • 진짜 배열 (map, filter 등 가능)
  • 화살표 함수에서도 사용 가능

객체(props)로 인자 전달하기

function f5(props) {
  if (props.a) console.log('props.a', props.a);
  console.log('props.b', props.b || '없음');
}

// f5();        // TypeError — props 가 undefined
f5({});         // 빈 객체라도 넣어야 안전
f5({ a: 10 }); // props.a 10
💡 React에서 props가 바로 이 패턴이다. 구조분해할당을 배우면 function f5({ a, b })처럼 더 깔끔하게 쓸 수 있다.

05함수 심화 — 표현식 · IIFE · 화살표 함수

JS에서 함수는 값(일급 객체)이다. 변수에 담거나, 인자로 넘기거나, 즉시 실행할 수 있다.

IIFE — 즉시 실행 함수

(function () {
  console.log('나는 존재한다');
})();  // 선언과 동시에 즉시 실행, 이름 없음

함수 표현식 — 콜백 패턴

const f = function (fn) {
  fn();  // 넘겨받은 함수를 실행 (콜백 패턴)
  console.log('나는 존재한다 2');
};

f(function () {
  console.log('포케 먹기 싫다');
});
💡 함수를 인자로 넘기는 것 = 콜백(Callback). setTimeout, addEventListener 등에서 매일 만나는 패턴이다.

화살표 함수 (Arrow Function)

(입력) => 출력 형태의 짧은 함수 문법. 본문이 한 줄이면 return과 { }를 생략할 수 있다.

const af  = () => 0;                         // 인자 없음, 한 줄 반환
const af2 = a => console.log(`결과: ${a}`);   // 인자 1개는 괄호 생략 가능

const af3 = (a) => {    // 여러 줄이면 { } 와 return 필요
  a++;
  a *= 2;
  return a;
};
console.log(af3(100)); // 202
✅ 화살표 함수는 자신만의 this를 갖지 않는다. 일반 함수와의 가장 큰 차이점이다.
구분함수 선언식함수 표현식 / 화살표
호이스팅✅ 됨 (선언 전 호출 가능)❌ 안 됨
this호출 방식에 따라 달라짐화살표: 상위 스코프 this
arguments✅ 사용 가능화살표: ❌ 불가

06구조분해할당 (Destructuring)

배열이나 객체에서 원하는 값을 꺼내 변수에 한 번에 담는 문법. 코드가 훨씬 짧아지고 읽기 쉬워진다.

객체 구조분해

const obj = { name: '윌리엄', age: 18, job: '반찬가게 직원' };

// 기본
const { name, age, job } = obj;

// 별칭(alias) — 다른 이름으로 받기
const { name: personName } = obj;  // name → personName

// 기본값 · 없는 키 · rest
const { salary, pet = '고양이', ...rest } = obj;
console.log(salary); // undefined — obj에 없음
console.log(pet);    // '고양이'  — 기본값
console.log(rest);   // { age: 18, job: '반찬가게 직원' }
❌ const { a } = undefined → TypeError! null/undefined에서 구조분해하면 에러가 난다.

배열 구조분해

const arr = ['삼성전자', 'SK하이닉스', 'SK스퀘어', '현대차', '삼성전기'];

const [a1, a2] = arr;       // 앞에서부터 순서대로
const [, , , , , a7] = arr; // 쉼표로 건너뛰기
const [b, ...c] = arr;      // 첫 번째 + 나머지 배열

// 변수 swap — 임시 변수 없이 한 줄 교환
let i = 0, j = 100;
[j, i] = [i, j];
console.log(i, j); // 100  0
💡 [b, a] = [a, b] 한 줄 swap은 정렬 알고리즘에서 자주 쓰이는 패턴이다.

재할당 vs 얕은복사 vs 깊은복사

const obj2 = obj;         // 재할당 — 같은 주소 공유
const obj3 = { ...obj };  // 얕은복사 — 새 객체, 1단계만 복사

// 주소 비교
console.log(obj === obj2); // true  — 같은 객체
console.log(obj === obj3); // false — 다른 객체

// 얕은복사의 함정: 내부 객체는 여전히 공유
arr[0] = { a: 1 };
const arr3 = [...arr];
arr[0].a = 1000;
console.log(arr3[0]); // { a: 1000 } — arr3 도 바뀜!
방식코드설명=== 비교
재할당const b = a주소 공유, 하나 바꾸면 둘 다 바뀜true
얕은복사{ ...a } / [...a]새 객체. 1단계는 독립, 중첩은 공유false
깊은복사structuredClone(a)모든 단계 완전 복사false
⚠ React에서 상태를 업데이트할 때 반드시 새 객체/배열을 만들어야 하는 이유가 바로 재할당은 주소를 공유하기 때문이다.

07호이스팅 (Hoisting)

JS 엔진이 코드를 실행하기 전에 var 선언과 function 선언을 파일 맨 위로 끌어올리는(hoist) 현상이다.

console.log(fun);    // 함수 객체 출력 — 선언 전인데 됨!
console.log(fun()); // 호출도 됨
console.log(bar);   // undefined — var는 선언만 올라감
// console.log(foo); // ReferenceError — let/const는 안 올라감

foo = 100;   // var 선언 없이 대입 (전역 오염 위험!)
var bar = 200;

function fun() { return 0; }
function fun() { return '덮어 씌웠지롱'; } // 재선언도 허용됨!
호이스팅 됨
  • function 선언식 → 통째로 위로 올라감
  • var → 선언만 올라감 (값은 undefined)
호이스팅 안 됨
  • let, const → ReferenceError
  • 화살표 함수 / 함수 표현식 → ReferenceError
✅ const / let과 화살표 함수(또는 함수 표현식)를 위주로 사용하면 호이스팅 문제를 겪을 일이 없다.

08스코프 (Scope)

스코프는 변수가 살 수 있는 범위다. var는 함수 스코프, let/const는 블록 스코프를 갖는다.

let v3 = '바깥 let';

{
  var v1 = '나는 var야';    // 블록 무시 — 밖에서 접근 가능!
  let v2 = '나는 let이야';  // 블록 안에서만 존재
  v3 = '안에서 바깥 수정';   // 바깥 변수는 수정 가능
}

console.log(v1); // '나는 var야'  ← 블록 밖에서 접근됨
// console.log(v2); // ReferenceError
console.log(v3); // '안에서 바깥 수정'

// for 루프의 i 도 블록 스코프
for (let i = 0; i < 5; i++) { ... }
// console.log(i); // ReferenceError

// var 는 함수 안에서만 격리
function f1() {
  var vv1 = '함수 안 var';
}
// console.log(vv1); // ReferenceError
키워드스코프블록 { }에서 격리호이스팅
var함수 스코프❌ (함수만 격리)✅ (undefined)
let블록 스코프✅❌ (에러)
const블록 스코프✅❌ (에러)
✅ var는 쓰지 말자. let / const만 사용하면 스코프 문제가 훨씬 단순해진다.

09클로저 (Closure)

클로저는 함수가 선언된 환경(스코프)을 기억하는 성질이다. 내부 함수가 외부 함수의 변수를 참조하면, 외부 함수가 종료된 후에도 그 변수가 메모리에 유지된다.

렉시컬 스코프 — 선언된 위치가 기준

const v = 'global';

function f() {
  console.log(v); // 'global' — 선언된 위치 기준
}

function f2() {
  const v = 'local';
  f();  // f 는 f2 안에서 호출되지만…
}

f2(); // 'global' 출력 — f 는 전역에서 선언됐기 때문!
💡 "어디서 호출하든, 선언된 시점의 스코프를 따른다" — 이것이 렉시컬(정적) 스코프다.

클로저 예시 — 카운터

function f3() {
  let c = 0;            // 이 변수가 클로저로 캡처됨
  return function () {
    return ++c;         // c 를 계속 기억하고 수정
  };
}

const counter = f3(); // f3 실행 종료 — 하지만 c 는 살아있음!
console.log(counter()); // 1
console.log(counter()); // 2  ← c 가 유지되고 있음
💡 JS는 접근 제어자(private)가 없다. 클로저로 변수를 함수 안에 가두면 외부에서 직접 접근 불가 → 상태 보호. React의 useState도 클로저 기반이다.

10객체 (Object)

객체는 여러 정보를 키: 값 쌍으로 묶어두는 자료구조다. 점(.) 또는 대괄호([])로 접근하며, 선언 후에도 자유롭게 속성을 추가·수정·삭제할 수 있다.

프로퍼티 추가 · 수정 · 삭제

const program = {
  name: '로또 생성기',
  numbers: [],
  pickNumber() {   // 메서드 단축 문법
    console.log('숫자를 뽑았습니다!');
  },
};

program.language = 'JS';        // 추가
program.numbers = [1,2,3,5,8]; // 수정
delete program.trash;           // 삭제 (거의 안 씀)

Object 유틸리티 메서드 3총사

Object.keys(program);    // ['name', 'language', 'numbers']
Object.values(program);  // ['로또 생성기', 'JS', [...]]
Object.entries(program); // [['name','로또 생성기'], ...]

// entries + 구조분해할당 — 키·값 동시에
for (const [k, v] of Object.entries(program)) {
  console.log(k, v);
}
메서드반환값주로 쓰는 상황
Object.keys(obj)키 배열키 목록만 필요할 때
Object.values(obj)값 배열값 목록만 필요할 때
Object.entries(obj)[키, 값] 쌍 배열키·값 동시에 필요할 때
💡 Object.entries + 구조분해할당 [k, v] 조합은 실무에서 자주 쓰이는 패턴이다.

11배열 메서드 — 스택 · 큐

push / pop / shift / unshift / splice는 배열의 앞/뒤에 요소를 추가·제거하는 기본 메서드다.

const arr = [1, 2, 3, 4, 5];

arr.push(11);      // 맨 뒤 추가  → [1,2,3,4,5,11]
arr.pop();         // 맨 뒤 제거  → 11 반환, [1,2,3,4,5]
arr.unshift('a'); // 맨 앞 추가  → ['a',1,2,3,4,5]
arr.shift();       // 맨 앞 제거  → 'a' 반환, [1,2,3,4,5]

// splice(시작인덱스, 개수) — 중간 제거
arr.splice(1, 2); // 인덱스 1부터 2개 제거 → [2,3] 반환
메서드위치동작반환값
push(val)맨 뒤추가새 길이
pop()맨 뒤제거제거된 요소
unshift(val)맨 앞추가새 길이
shift()맨 앞제거제거된 요소
splice(i, n)중간제거/교체제거된 요소 배열
스택 (Stack) — 후입선출 LIFO
  • push로 넣고 pop으로 뺌
  • 마지막에 넣은 게 먼저 나옴
큐 (Queue) — 선입선출 FIFO
  • push로 넣고 shift로 뺌
  • 먼저 넣은 게 먼저 나옴
⚠ push / pop / shift / unshift / splice 는 모두 원본 배열을 직접 수정한다. React에서는 [...arr]로 새 배열을 만들어 사용한다.

전체 핵심 요약

01 반복문

while / for. break로 종료, continue로 건너뜀. 배열 조건은 < arr.length

02 for-in / for-of

in은 키 꺼내기, of는 값 꺼내기. 일반 객체에 of 직접 사용 불가

03 함수 기본

지역변수는 밖으로 못 나옴. return으로 꺼내고 종료. early return 패턴

04 매개변수

기본값, ...rest로 가변 인자 처리. 객체 하나로 묶어 전달 (props 패턴)

05 함수 심화

함수는 값(일급 객체). IIFE, 콜백, 화살표 함수. 화살표 함수는 this 없음

06 구조분해할당

객체 { }, 배열 [ ]에서 값 꺼내기. swap, rest, 얕은복사 vs 깊은복사

07 호이스팅

var / function은 위로 올라감. const / let + 화살표 함수로 방지

08 스코프

var는 함수 스코프, let / const는 블록 스코프. var는 쓰지 말 것

09 클로저

함수는 선언된 위치의 스코프를 기억. 상태 보호 & 유지에 사용. React useState의 기반

10 객체

프로퍼티 추가/수정/삭제. Object.keys / values / entries로 순회

11 배열

push/pop(스택 LIFO), push/shift(큐 FIFO), splice(중간 수정). 원본 수정에 주의!
